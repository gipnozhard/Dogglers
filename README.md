# Проект:Приложение для догглеров

В этой кодлаборатории представлено новое приложение под названием Dogglers, которое вы создадите самостоятельно. В этой кодлаборатории описаны шаги по завершению проекта приложения Dogglers, включая настройку проекта и тестирование в Android Studio.

Эта кодовая лаборатория отличается от других в этом курсе. В отличие от предыдущих кодовых лабораторий, целью этой кодовой лаборатории является не предоставление пошагового руководства по созданию приложения. Вместо этого эта кодовая таблица предназначена для настройки проекта, который вы будете выполнять самостоятельно, предоставляя вам инструкции о том, как завершить приложение и проверить свою работу самостоятельно.

Вместо кода решения мы предоставляем набор тестов как часть приложения, которое вы скачаете. Вы запустите эти тесты в Android Studio (мы покажем вам, как это сделать позже в этой кодовой лаборатории) и посмотрите, проходит ли ваш код. Это может занять несколько попыток — даже профессиональным разработчикам редко удается пройти все тесты с первого раза! После того, как ваш код пройдет все тесты, вы можете считать этот проект завершенным.

Мы понимаем, что вы, возможно, просто хотите проверить это решение. Мы намеренно не предоставляем код решения, потому что хотим, чтобы вы на практике узнали, каково это - быть профессиональным разработчиком. Это может потребовать от вас использования различных навыков, с которыми у вас еще нет большой практики, таких как:

 ## 2. Обзор приложения

Добро пожаловать в проект: приложение Dogglers!

В Google мы ласково называем наших коллег гуглерами. Поскольку многие гуглеры держат собак в качестве домашних животных, мы подумали, что было бы забавно создать приложение для наших друзей-собак под названием Dogglers. Ваша задача - внедрить Dogglers, который показывает прокручивающиеся списки домашних собак Google вместе с небольшой информацией о каждой из них, такой как их имя, возраст, хобби и фотография. В этом проекте вы создадите макеты для элементов RecyclerView в приложении Dogglers и реализуете адаптер, чтобы список собак можно было представить тремя способами: горизонтальной прокруткой, вертикальной прокруткой и макетом сетки с вертикальной прокруткой.

Когда вы запускаете приложение, вам предоставляются варианты горизонтального, вертикального расположения и сетки.

![image](https://github.com/gipnozhard/Dogglers/assets/71705375/f9825483-08cb-4f73-ba08-b1fad05fbad3)

Первый вариант - это режим вертикальной прокрутки с элементами, занимающими всю ширину экрана.

![image](https://github.com/gipnozhard/Dogglers/assets/71705375/37f8dc78-7bd2-4884-9374-465fcede7c22)

Второй вариант показывает список собак в режиме рециркулятора с горизонтальной прокруткой.

![image](https://github.com/gipnozhard/Dogglers/assets/71705375/a5f15600-da41-4e70-bd98-c98ae5136f50)

Третий вариант показывает собак в виде сетки с вертикальной прокруткой, где в каждой строке показаны две собаки.

![image](https://github.com/gipnozhard/Dogglers/assets/71705375/b821521c-e43b-4bb4-ad24-e834c3ac92d6)

Все эти макеты поддерживаются одним и тем же классом адаптера. Вашей задачей будет создать макеты для карточек просмотра recycler, а затем внедрить адаптер таким образом, чтобы каждый элемент был заполнен информацией о каждой собаке.

### Реализовать макет

Вертикальный и горизонтальный макеты идентичны, поэтому вам нужно всего лишь реализовать один файл макета для обоих. Макет сетки отображает всю ту же информацию, но кличка, возраст и увлечения собаки расположены вертикально, поэтому для этого случая вам понадобится отдельный макет. Для обоих макетов требуется четыре разных вида отображения информации о каждой собаке.

 1. Приложение ImageView с фотографией собаки

 2. A TextView с именем собаки

 3. A TextView с учетом возраста собаки

 4. A TextView с увлечениями собаки

Вы также заметите некоторый стиль на каждой карточке для отображения границы и тени. Этим занимается MaterialCardView, который уже добавлен в файлы макета в стартовом проекте. Внутри каждого из них MaterialCardView есть ConstraintLayout поле, куда вам нужно будет добавить оставшиеся просмотры.

--------------------------------------------------------------
Совет: Вы можете использовать любой идентификатор по вашему выбору для каждого представления, но имейте в виду, что оба этих макета будут использовать один и тот же ViewHolder класс, поэтому вам нужно убедиться, что соответствующие представления в каждом макете используют один и тот же идентификатор. Например, как макет сетки, так и горизонтальный / вертикальный макет будут иметь TextView с идентификатором dog_name.
--------------------------------------------------------------

Для реализации макетов вам потребуется поработать с двумя XML-файлами: vertical_horizontal_list_item.xml для горизонтального и вертикального макетов и grid_list_item.xml для макета сетки.

 ### Внедрить адаптер

После того, как вы определили свои макеты, вашей следующей задачей будет внедрение RecyclerView адаптера. Откройте приложение DogCardAdapter.kt в пакете adapter. Вы увидите множество TODO комментариев, которые помогут объяснить, что вам нужно реализовать.

![image](https://github.com/gipnozhard/Dogglers/assets/71705375/b357610c-457f-4ad5-93a6-41c643c3a188)

Для внедрения адаптера вам потребуется выполнить пять шагов.

 1. Определите переменную или константу для списка данных о собаках. Список можно найти в пакете data в объекте с именем DataSource, и он выглядит следующим образом:

        object DataSource {
        
           val dogs: List<Dog> = listOf( ...
        
        }

dogs Свойство имеет тип List<Dog>. Dog Класс находится в пакете model и определяет четыре свойства: изображение (представленное идентификатором ресурса) и три String свойства.

    data class Dog(
       @DrawableRes val imageResourceId: Int,
       val name: String,
       val age: String,
       val hobbies: String
    )

Установите переменную, которую вы определяете в DogCardAdapter, в dogs список в DataSource объекте.

 2. РеализуйтеDogCardViewHolder. Владелец вида должен привязать четыре вида, которые необходимо установить для каждой карточки просмотра recycler. Один и тот же держатель вида будет использоваться совместно для grid_list_item и vertical_horizontal_list_item макетов, поскольку все виды являются общими для обоих макетов. DogCardViewHolder Должно включать свойства для следующих идентификаторов представления: dog_image, dog_name, dog_age и dog_hobbies.

 3. В onCreateViewHolder() вы хотите либо увеличить макет, grid_list_item либо vertical_horizontal_list_item. Как вы узнаете, какой макет использовать? В определении адаптера вы можете видеть, что при создании экземпляра адаптера передается значение, называемое layout of type Int.

        class DogCardAdapter(
           private val context: Context?,
           private val layout: Int
        ): RecyclerView.Adapter<DogCardAdapter.DogCardViewHolder>() {

Это соответствует значению, определенному в Layout объекте, расположенном в const пакете.

    object Layout {
       val VERTICAL = 1
       val HORIZONTAL = 2
       val GRID = 3
    }

Значение layout будет равно 1, 2 или 3, но вы можете проверить его по идентификаторам VERTICAL, HORIZONTAL и GRID из Layout объекта.

Для GRID макета увеличьте grid_list_item макет, а для HORIZONTAL и VERTICAL макетов увеличьте vertical_horizontal_list_item макет. Метод должен возвращать DogCardViewHolder экземпляр, представляющий раздутый макет.

-----------------------------------------------------------
Совет: Вы можете использовать условный оператор, например if или when, чтобы задать переменную на основе типа макета (grid_list_item для GRID и vertical_horizontal_list_item для VERTICAL или HORIZONTAL). Как только у вас будет правильный идентификатор макета, просто передайте его в метод для расширения макета.
-----------------------------------------------------------

 4. РеализуйтеgetItemCount(), чтобы возвращать длину списка собак.

 5. Наконец, вам нужно реализовать onBindViewHolder() установку данных в каждой из карточек просмотра recycler. Используйте position для доступа к правильным данным о собаке из списка, а также для установки изображения и клички собаки. Используйте строковые ресурсы, dog_age и dog_hobbies, чтобы соответствующим образом отформатировать возраст и увлечения.

----------------------------------------------------------
Совет: В onBindViewHolder() методе мы определили resources переменную, которую вы можете использовать для ссылки на строковые ресурсы, не используя context?.resources каждый раз.
